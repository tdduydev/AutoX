# ML / AutoML

### 2. Train Model

Train a machine learning model with your own data.

#### Step-by-Step
1. Switch to the **Train Model** tab
2. **Select Algorithm** — Choose from the 12 available algorithms
3. **Upload Data** — Upload a CSV file with your training data
4. **Configure Features**:
   - Select **input columns** (features)
   - Select **target column** (label/output)
   - Set **train/test split** ratio (default: 80/20)
5. **Set Hyperparameters** — Algorithm-specific settings
6. Click **Train** — Monitor training progress
7. View **Results**:
   - Accuracy / R² score
   - Confusion matrix (classification)
   - Feature importance
   - Validation metrics

---

### 3. AutoML

Automatic model selection and hyperparameter tuning.

#### Step-by-Step
1. Switch to the **AutoML** tab
2. Upload your dataset (CSV)
3. Select the **target column**
4. Choose the **task type** (classification or regression)
5. Click **Run AutoML**
6. The system automatically:
   - Tries multiple algorithms
   - Tunes hyperparameters
   - Cross-validates results
   - Ranks models by performance
7. Review the **leaderboard** — best model appears on top
8. Click **Deploy** on the best model to save it to the registry

---

### 4. Model Registry

View and manage all trained models.

| Field | Description |
|-------|-------------|
| **Model Name** | Custom name or auto-generated |
| **Algorithm** | Which algorithm was used |
| **Accuracy** | Model performance metric |
| **Created** | Training timestamp |
| **Status** | Active / Inactive |
| **Actions** | Download, delete, deploy |

## Tips

- Start with simpler models (Linear/Logistic Regression) before trying ensemble methods
- Ensure your data is clean — remove null values and outliers before training
- Use AutoML for first-pass model selection, then fine-tune manually
- Compare at least 3 algorithms on the same dataset
- Use cross-validation for more reliable accuracy estimates
